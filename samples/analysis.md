# Shogi-TransMaster: ASR 誤認識の分析と LLM による補正戦略
# Shogi-TransMaster: ASR 识别错误分析与 LLM 纠错策略

## 1. 現状の課題 (Current Issues)
**音声認識 (ASR) の限界 / 语音识别的局限性**
汎用的な Whisper モデル（large-v3）では、将棋解説特有のノイズ（駒の音、解説者の速い口調）により、専門用語が日常単語として誤認識されるケースが多々あります。
通用 Whisper 模型（large-v3）在面对将棋解说特有的环境（棋子碰撞音、解说员语速快）时，经常会将专业术语误识别为日常词汇。

* **例 (Example):** * 認識結果 (ASR Result): 「14部」 or 「イチヨンブ」
    * 正解 (Ground Truth): 「1四歩」

## 2. 解決策：API へのキーワード注入 (Proposed Solution)
**キーワードヒントによるセマンティック補正 / 通过关键词提示进行语义校正**
ASR 自体の再学習（Fine-tuning）は計算コストが高いため、本システムでは **API 翻訳呼び出し時に専門用語のキーワードをコンテキストとして注入する** 手法を採用しました。
由于 ASR 底层微调成本较高，本系统采用在 **API 调用翻译时，将专业术语作为“关键词提示（Context）”注入** 的方案。

### 仕組み (Mechanism):
1.  `shogi_glossary.json` から現在の文脈に関連する用語を抽出。
2.  LLM（DeepSeek / Gemini）に対し、「これらのキーワードが含まれている可能性が高い」というヒントを Prompt に含める。
3.  LLM が ASR の誤字（例：77金）を、将棋の知識に基づいて正しい形式（7七金）に「推論」して修正。

## 3. 比較実験の考察 (Comparison & Discussion)
`samples/` 内の `raw_baseline_whisper.vtt` と `refined_shogi_output.vtt` を比較すると、以下の改善が確認できました。

* **座標の正規化**: 単なる数字の羅列から、将棋標準の記譜法への変換。
* **ホモフォンの解消**: 「同飛（どうひ）」が「動詞」と誤認識されても、ヒント注入により LLM が正しい漢字を選択。
* **术语纠错**: 即使 ASR 将“同飞”听成“动词”，LLM 在关键词提示下也能精准修正。

## 4. 今後の展望 (Future Work)
現在の「API 段階での補正」は、ASR が全く意味を成さないレベルで誤認識した場合には限界があります。
今後は、広島大学での研究を通じ、ASR モデル（Whisper）自体のドメイン適応（LoRA 等を用いた微調整）を行い、入力段階での精度向上を目指します。
目前的“API 纠错”在 ASR 彻底识别错误时仍有局限。未来计划在广岛大学期间，尝试利用 LoRA 等技术对 Whisper 进行领域微调，从源头提升精度。
